# Security policy

## Reporting a vulnerability

Please do not open a public issue for an undisclosed vulnerability. Report it through [GitHub's private vulnerability reporting](https://github.com/lucastraba/ai-canon/security/advisories/new). Include the affected version, impact, reproduction steps, and any suggested mitigation. You can expect an acknowledgement within 72 hours.

## Trust model

An ai-canon repository is trusted code. It distributes agent instructions, MCP commands, environment-backed configuration, and helper scripts. Protect write access to the canon and review its changes before syncing them across repositories.

The CLI confines manifest inputs and generated destinations, marks owned files, refuses unowned root configuration, preflights changes, and skips unresolved secrets. These controls limit accidents and compromise impact; they do not make an untrusted canon safe to use.

Generated MCP configuration can contain resolved credentials. Keep `.ai/`, `.ai.local/`, `.ai.lock.json`, agent configuration directories, `.mcp.json`, and `opencode.json` untracked as generated by `ai-canon init`. Prefer Git credential helpers over credentials embedded in canon URLs.

## Supported versions

Security fixes are provided in the latest npm release. Upgrade before reporting an issue that is already fixed on `main`.
